## 1. 配置模型概述

ARP-Rust 分为两端：

- **服务端（arps）**：运行在公网服务器，接收客户端控制连接和工作连接，暴露远程端口或 vhost 路由
- **客户端（arpc）**：运行在私有网络机器上，连接到服务端，将本地流量透出到公网

典型启动方式：

```bash
./arps -c server.toml
./arpc -c client.toml
```

## 2. 服务端常用字段

最简服务端配置：

```toml
bind_addr = "0.0.0.0"
bind_port = 17000
log_level = "info"

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"

[[allow_ports]]
start = 6000
end = 7000
```

### 字段说明

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `bind_addr` | 服务端监听地址 | `0.0.0.0` |
| `bind_port` | TCP/WS 控制端口 | — |
| `kcp_bind_port` | KCP UDP 端口（可选，默认同 `bind_port`） | — |
| `quic_bind_port` | QUIC UDP 端口（可选，默认同 `bind_port`） | — |
| `vhost_http_port` | HTTP vhost 公网端口（0 表示禁用） | `0` |
| `vhost_https_port` | HTTPS vhost 公网端口（0 表示禁用） | `0` |
| `subdomain_host` | HTTP/HTTPS 子域名路由的根域名 | `""` |
| `dashboard_addr` / `dashboard_port` | 管理面板地址和端口（可选） | — |
| `allow_ports` | 允许客户端申请的端口范围（TCP/UDP 代理） | — |
| `log_level` | 日志级别：`trace/debug/info/warn/error` | `info` |
| `log_file` | 日志文件路径前缀（不填则输出到 stdout） | `""` |
| `log_max_days` | 日志文件保留天数 | `1` |
| `max_pool_count` | 每个客户端允许的最大预热连接数（0=不限） | `0` |
| `transport.tcp_mux` | 服务端是否开启 TCP 多路复用 | `true` |
| `transport.heartbeat_timeout` | 心跳超时秒数（客户端无响应多久后断开） | `90` |

## 3. 客户端常用字段

最简客户端配置：

```toml
server_addr = "server.example.com"
server_port = 17000
client_id = "my-node-1"
log_level = "info"

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"
pool_count = 1
```

### 字段说明

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `server_addr` | 服务端公网地址 | — |
| `server_port` | 服务端控制端口 | — |
| `client_id` | 客户端唯一标识（用于代理续接） | 自动生成 |
| `admin_addr` / `admin_port` | 本地管理 API 地址和端口 | — |
| `admin_user` / `admin_pwd` | 管理 API Basic Auth（可选） | — |
| `log_level` | 日志级别 | `info` |
| `log_file` | 日志文件路径前缀 | `""` |
| `log_max_days` | 日志文件保留天数 | `1` |
| `transport.protocol` | 传输协议：`tcp/kcp/quic/websocket` | `tcp` |
| `transport.tcp_mux` | 是否开启 TCP 多路复用（仅对 tcp 类型代理生效） | `true` |
| `transport.pool_count` | 预热工作连接数量 | `1` |
| `transport.heartbeat_interval` | 心跳发送间隔（秒） | `30` |
| `transport.heartbeat_timeout` | 心跳超时时间（秒） | `90` |

### 关于 `client_id`

`client_id` 是客户端的稳定身份标识：

- 同一个 `client_id` 的客户端重连后，可以接管旧连接上已注册的同名代理（takeover），不会报"代理名冲突"
- 不同 `client_id` 的客户端注册同名代理时，服务端会拒绝后者（保护先注册的客户端）
- 建议每个物理节点分配一个固定的 `client_id`

### 关于 `tcp_mux`

`tcp_mux = true` 开启 TCP 多路复用：允许一条工作连接同时承载多个用户连接，减少连接握手开销。

**适用范围**：只对 `type = "tcp"` 的代理生效。`http`/`https` 代理不受此参数影响（它们直接透传 HTTP 字节流）。

**默认值**：`true`（如配置文件中不写，等同于开启）。

### 关于 `pool_count`

`pool_count` 设置预热的工作连接数量：客户端启动后提前建立指定数量的工作连接等待复用，用户请求到来时无需等待新连接握手。

- `pool_count = 0`：不预热，每次按需建连
- `pool_count = 1`：预热 1 条（推荐默认值）
- 较大值适合高并发场景

## 4. 代理字段说明

```toml
[[proxies]]
name = "ssh"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 22
remote_port = 6001
```

### 通用字段

| 字段 | 说明 |
|------|------|
| `name` | 代理名称，部署范围内需唯一 |
| `type` | 代理类型：`tcp/udp/http/https/stcp/sudp/xtcp` |
| `local_ip` | 本地服务 IP（通常是 `127.0.0.1`） |
| `local_port` | 本地服务端口 |
| `remote_port` | 在服务端分配的公网端口（TCP/UDP 类型使用） |
| `custom_domains` | 自定义域名列表（`http`/`https` 使用） |
| `subdomain` | 子域名路由（`http`/`https` 使用，需配合服务端 `subdomain_host`） |
| `sk` | 共享密钥（`stcp`/`sudp`/`xtcp` 使用） |
| `fallback_to_relay` | XTCP 打洞失败时是否降级到中继 TCP | 
| `use_encryption` | 是否加密数据（AES-GCM，仅对 UDP/STCP 等类型有效） |
| `use_compression` | 是否压缩数据 |
| `multiplexer` | 通常留空，`tcp_mux=true` 时自动为 tcp 类型设置 |

## 5. 传输协议场景

### 5.1 TCP 明文（本地/实验室网络）

适用场景：
- 最简单的部署方式
- 内网/受控网络，不需要加密
- 快速验证功能

```toml
# server.toml
bind_addr = "0.0.0.0"
bind_port = 17000

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"
tcp_mux = true

[[allow_ports]]
start = 6000
end = 7000
```

```toml
# client.toml
server_addr = "server.example.com"
server_port = 17000
client_id = "node-1"

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"
tcp_mux = true
pool_count = 1

[[proxies]]
name = "ssh"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 22
remote_port = 6001
```

参考示例：`examples/server.toml`、`examples/client.toml`

---

### 5.2 TCP + TLS（公网加密隧道）

适用场景：
- 需要传输加密，不需要 WebSocket 包装
- 网络环境允许直接 TCP（非 HTTP 代理环境）

```toml
# server.toml
bind_addr = "0.0.0.0"
bind_port = 17000

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"

[transport.tls]
enable = true
cert_file = "/etc/arp/server.crt"
key_file = "/etc/arp/server.key"

[[allow_ports]]
start = 6001
end = 7000
```

```toml
# client.toml
server_addr = "your.server.name"
server_port = 17000
client_id = "tls-node-1"

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"
pool_count = 1

[transport.tls]
enable = true
trusted_ca_file = "/etc/arp/server.crt"
server_name = "your.server.name"  # 若 server_addr 是 IP 则必填

[[proxies]]
name = "ssh_tls"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 22
remote_port = 6001
```

> **注意**：`server_name` 必须与服务端证书的 SAN/CN 匹配。

## 6. 应用场景示例

### 6.1 暴露 SSH

最常见的用途，将私网机器的 SSH 暴露到公网。

```toml
[[proxies]]
name = "ssh"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 22
remote_port = 6001
```

连接方式：

```bash
ssh user@server.example.com -p 6001
```

协议选择：
- 内网/实验室：`tcp`
- 公网生产：`wss`（推荐）
- UDP 可用且追求性能：`quic`

---

### 6.2 暴露本地 Web 服务（TCP 端口转发）

直接端口转发，不需要域名路由。

```toml
[[proxies]]
name = "web_tcp"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 8080
remote_port = 6002
```

访问方式：`http://server.example.com:6002`

---

### 6.3 HTTP 虚拟主机路由（按域名分发）

多个客户端的 Web 服务共享服务端同一个 HTTP/HTTPS 端口，通过 `Host` 头或 TLS SNI 路由。

服务端必须配置：

```toml
# server.toml
vhost_http_port = 80       # HTTP vhost 公网端口
vhost_https_port = 443     # HTTPS vhost 公网端口（可选）
subdomain_host = "example.com"  # 子域名根域（可选）
```

**使用自定义域名（推荐）**：

```toml
# client.toml
[[proxies]]
name = "my_web"
type = "http"
local_ip = "127.0.0.1"
local_port = 8080
custom_domains = ["app.example.com"]
```

访问：`http://app.example.com`（服务端 vhost_http_port 为 80）

**使用子域名路由**：

```toml
[[proxies]]
name = "my_web"
type = "http"
local_ip = "127.0.0.1"
local_port = 8080
subdomain = "app"
```

访问：`http://app.example.com`（需服务端配置 `subdomain_host = "example.com"`）

**HTTPS 自定义域名**：

```toml
[[proxies]]
name = "my_web_https"
type = "https"
local_ip = "127.0.0.1"
local_port = 8443
custom_domains = ["app.example.com"]
```

参考示例：
- `examples/server_vhost_http.toml`
- `examples/server_vhost_https.toml`
- `examples/client_http_custom_domain.toml`
- `examples/client_http_subdomain.toml`
- `examples/client_https_custom_domain.toml`

> **重要**：`http`/`https` 代理不受 `tcp_mux` 影响，无论 `tcp_mux` 是否开启，HTTP 流量始终走原始字节流透传路径。

## 7. XTCP NAT 穿透

当双端都在 NAT 后面时，XTCP 可以尝试直接 P2P 打洞，打洞失败时可降级到中继 TCP。

**提供方（provider）**：

```toml
server_addr = "your.server.name"
server_port = 17000
client_id = "xtcp-provider-1"

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"

[[proxies]]
name = "xtcp_demo"
type = "xtcp"
local_ip = "127.0.0.1"
local_port = 22
sk = "xtcp_shared_secret"
fallback_to_relay = true  # 打洞失败时降级到中继
```

**访问方（visitor）**：

```toml
server_addr = "your.server.name"
server_port = 17000
client_id = "xtcp-visitor-1"

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "tcp"

[[visitors]]
name = "xtcp_visitor"
type = "xtcp"
server_name = "xtcp_demo"       # 对应 provider 的 proxy name
sk = "xtcp_shared_secret"
bind_addr = "127.0.0.1"
bind_port = 6001                 # 本地监听端口，连此端口等同于连 provider
fallback_to_relay = true
xtcp_punch_timeout_secs = 12
```

连接方式：

```bash
ssh user@127.0.0.1 -p 6001
```

参考示例：`examples/client_xtcp_provider.toml`、`examples/client_xtcp_visitor.toml`

## 8. TCP 负载均衡组

多个客户端节点共享同一个公网 TCP 端口，服务端轮询分发流量。

要求：
- 每个节点使用 `type = "tcp"` 或 `"stcp"`
- 必须使用**固定** `remote_port`（不能为 0）
- 同组节点使用相同的 `load_balancer.group`

节点 A：

```toml
[[proxies]]
name = "api-node-a"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 8080
remote_port = 6010

[proxies.load_balancer]
group = "api-cluster"
group_key = "v1"
```

节点 B（另一台客户端机器）：

```toml
[[proxies]]
name = "api-node-b"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 8080
remote_port = 6010

[proxies.load_balancer]
group = "api-cluster"
group_key = "v1"
```

- 服务端监听 `6010` 端口，轮询分发给所有健康节点
- 节点健康检查失败时自动临时摘除，恢复后重新加入

## 9. TLS 证书规则

当使用 `transport.tls.enable = true` 或 `protocol = "quic"` 时：

**服务端**：
- 必须设置 `cert_file` 和 `key_file`
- 证书的 SAN（subjectAltName）必须包含客户端访问的域名或 IP

**客户端**：
- 设置 `trusted_ca_file`（信任的 CA 证书）
- 当 `server_addr` 是 IP，或者需要明确指定证书验证名称时，必须设置 `server_name`

生成自签名证书：

```bash
openssl req -x509 -newkey rsa:2048 -nodes \
  -keyout server.key \
  -out server.crt \
  -days 365 \
  -subj "/CN=your.server.name" \
  -addext "subjectAltName=DNS:your.server.name,IP:1.2.3.4" \
  -addext "basicConstraints=CA:FALSE"
```

或使用内置脚本：

```bash
cd examples
bash gen_self_signed_cert.sh your.server.name 1.2.3.4
```

## 10. 协议选择建议

| 场景 | 推荐协议 |
|------|---------|
| 本地实验室 / 内网 | `tcp` |
| 公网生产部署（首选） | `wss`（WebSocket + TLS） |
| UDP 可用，追求性能 | `quic` |
| 受控网络 UDP 传输 | `kcp` |
| 域名路由 Web 发布 | `http`/`https` 代理 + vhost 端口 |
| NAT 穿透 | `xtcp` |

**tcp_mux 参数是否影响协议选择？**
- 不影响。`tcp_mux` 和传输协议（tcp/ws/kcp/quic）是独立的两个维度。
- `tcp_mux` 只对 `type = "tcp"` 的代理生效，`http`/`https` 代理永远不走 mux 路径。

## 11. 常见错误

- 忘记修改 `auth.token`（使用了示例中的占位符）
- `remote_port` 不在服务端 `allow_ports` 范围内
- 使用 `websocket + tls` 或 `quic` 时未正确配置证书/私钥/CA
- `server_addr` 是 IP 地址时未设置 `transport.tls.server_name`
- 多个客户端使用了相同的 `proxy.name` 但 `client_id` 不同（导致后者被拒绝）
- TCP 负载均衡组没有固定 `remote_port`（设为 0 会报错）
- `http`/`https` 代理没有配置 `custom_domains` 或 `subdomain`
- 服务端 `vhost_http_port` 为 0（默认禁用），导致 `http` 代理注册失败

## 12. 推荐工作流程

1. 先用 `examples/server.toml` + `examples/client.toml` 验证一个简单 TCP 代理
2. 确认基础联通后，切换到 `wss` 传输（公网部署）
3. 按需添加 HTTP vhost、QUIC 等高级功能
4. 为每个部署节点分配固定的 `client_id`
5. 生产环境开启 `log_file` 记录日志，配置 `dashboard_port` 监控

## 13. 日志文件输出与轮转

默认情况下 `arps` 和 `arpc` 将日志输出到 **stdout**。

要将日志写入文件并按天自动轮转，在配置文件中添加：

```toml
# 日志文件路径前缀
# 实际文件名为 <前缀>.<YYYY-MM-DD>，每天自动轮转
# 目录不存在时会自动创建
log_file = "/var/log/arp/arps"

# 日志文件保留天数
# 启动时自动删除超过此天数的旧日志
# 默认值：1（只保留今天的日志）
# 设为 0 禁用自动清理
log_max_days = 7
```

设置 `log_file` 后：
- 日志**仅**写入文件，stdout 不再输出
- 文件名格式为 `<前缀>.<YYYY-MM-DD>`，例如 `arps.2026-04-21`
- 每天零点自动创建新文件
- 文件输出中 ANSI 颜色代码会自动去除

服务端示例（含文件日志）：

```toml
bind_addr = "0.0.0.0"
bind_port = 17000
log_level = "info"
log_file = "/var/log/arp/arps"
log_max_days = 7

[auth]
method = "token"
token = "替换为你的密钥"

[transport]
protocol = "websocket"
tcp_mux = true

[transport.tls]
enable = true
cert_file = "/etc/arp/server.crt"
key_file  = "/etc/arp/server.key"

[[allow_ports]]
start = 6001
end = 7000
```

客户端示例（含文件日志）：

```toml
server_addr = "your.server.name"
server_port = 443
client_id   = "prod-node-1"
log_level   = "info"
log_file    = "/var/log/arp/arpc"
log_max_days = 7

[auth]
method = "token"
token  = "替换为你的密钥"

[transport]
protocol   = "websocket"
tcp_mux    = true
pool_count = 1

[transport.tls]
enable          = true
trusted_ca_file = "/etc/arp/server.crt"
server_name     = "your.server.name"

[[proxies]]
name       = "ssh"
type       = "tcp"
local_ip   = "127.0.0.1"
local_port = 22
remote_port = 6001
```

> `log_file` 填路径**前缀**，不要加 `.log` 后缀，日期后缀由程序自动追加。

## 14. 客户端管理 API 与动态代理管理

在客户端配置中添加 `admin_port`，启用本地 HTTP 管理接口：

```toml
admin_addr = "127.0.0.1"
admin_port = 7400
admin_user = "admin"    # 可选，Basic Auth 用户名
admin_pwd = "secret"    # 可选，Basic Auth 密码
```

### 可用接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/status` | 连接状态、服务端信息、代理列表 |
| GET | `/api/v1/proxies` | 列出所有已注册代理 |
| GET | `/api/v1/proxies/:name` | 代理详情和健康状态 |
| POST | `/api/v1/proxies` | 动态添加代理（JSON body） |
| DELETE | `/api/v1/proxies/:name` | 动态删除代理 |
| POST | `/api/v1/reload` | 重新加载配置文件，差量应用变更 |

### 动态添加代理

```bash
curl -u admin:secret -X POST http://127.0.0.1:7400/api/v1/proxies \
  -H "Content-Type: application/json" \
  -d '{"name":"new-svc","type":"tcp","local_ip":"127.0.0.1","local_port":3000,"remote_port":6002}'
# 返回: {"ok":true,"message":"proxy registered","remote_addr":"0.0.0.0:6002"}
```

### 动态删除代理

```bash
curl -u admin:secret -X DELETE http://127.0.0.1:7400/api/v1/proxies/new-svc
# 返回: {"ok":true,"message":"proxy new-svc closed"}
```

### 热重载配置

```bash
curl -u admin:secret -X POST http://127.0.0.1:7400/api/v1/reload
# 返回: {"ok":true,"message":"reload complete: 1 added, 0 removed"}
```

## 15. CLI 诊断命令

`arpc` 支持以下子命令（需要配置 `admin_port`）：

```bash
# 启动客户端（默认）
arpc -c client.toml
arpc run -c client.toml

# 查看连接状态和代理列表
arpc status -c client.toml

# 诊断指定代理（检查控制连接、代理注册、本地服务）
arpc check ssh -c client.toml

# 诊断所有代理
arpc check -c client.toml
```

## 16. SIGHUP 热重载配置

在 Unix 系统上，发送 SIGHUP 信号可以让客户端重新加载配置文件：

```bash
kill -HUP $(pidof arpc)
```

重载行为：
- 新增的代理自动注册
- 已删除的代理自动注销
- 未变化的代理不受影响，活跃连接不中断

## 17. HMAC 认证

从 v0.4.0 起，Token 不再明文传输。

- 客户端计算 `HMAC-SHA256(token, timestamp)` 作为认证签名
- 服务端对所有已知 Token 验证签名
- 强制 5 分钟防重放窗口（时间戳偏差超过 300 秒的消息直接拒绝）

无需任何额外配置，双端都是 v0.4.0+ 时自动生效。

> **注意**：v0.4.0 客户端与旧版服务端不兼容，反之亦然。

## 19. 优雅退出

服务端和客户端均支持 `Ctrl+C` / `SIGTERM` 优雅退出：

- **服务端**：停止接受新连接，等待活跃连接完成（最多 30 秒），然后退出
- **客户端**：正常断开与服务端的连接后退出

## 21. SSE 实时事件流

服务端提供 Server-Sent Events 实时监控端点：

```bash
curl -N http://server:17500/api/v1/events/stream
```

事件格式：

```
data: {"event":"client_login","client_id":"user@host","peer_addr":"...","run_id":"..."}

data: {"event":"proxy_registered",...}
```

每隔一段时间发送心跳注释，防止连接超时。

## 22. 多租户运行时限制

通过 `[[auth.rules]]` 为每个 Token 配置独立的运行时限制：

```toml
[[auth.rules]]
token = "team-frontend"
allow_proxy_types = ["http", "https"]   # 只允许注册 http/https 类型代理
allow_subdomain_prefixes = ["fe-"]      # 子域名必须以 fe- 开头
max_pool_count = 5                      # 最大预热连接数
max_connections = 100                   # 最大并发连接数（0=不限）
bandwidth_limit_bytes = 10485760        # 带宽上限 10 MB/s（0=不限）
```
